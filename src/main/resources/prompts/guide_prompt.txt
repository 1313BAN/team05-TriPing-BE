아래 JSON 형식에 맞춰 관광지에 대한 상세한 가이드를 작성해줘.
각 필드는 단순 요약이 아니라, 실제로 현장에서 사용자가 읽고 유용하게 느낄 수 있도록 구체적이고 풍부하게 작성해야 해.
현장감 있고 실질적인 정보 위주로, 단순 홍보성 문구가 아니라 실제 경험에 기반한 느낌으로 작성해줘.
그리고 가장 중요한 점은 각 항목에 대해 **정보가 없으면 반드시 null 값으로 응답해야 해.**
억지로 채우지 말고 잘 모르겠으면 null로 응답해줘. 가장 중요한 점 : 값으로 null을 넣지 말고 실제 null 값을 넣어줘.

절대 문자열 "null"을 넣지 말고, 반드시 실제 JSON의 null 값을 넣어야 해.
   - 예: "holiday": null ← (O), "holiday": "null" ← (X)
JSON 형식으로 null 작성할 때 다음과 같은 실수를 절대로 해서는 안돼.
   -  "a : null" → 이건 절대 안 됨.
   -  "a": null → 이렇게 정확히 넣어야 해.

관광지 이름: %s
주소: %s

아래 항목들을 JSON으로 작성해줘:

- "description":
  장소를 설명하는 소개 텍스트를 리스트(String 배열)로 작성해줘.
  단순한 요약이 아니라, 주변 풍경, 건물의 특징, 분위기, 사람들의 반응, 계절별 매력 등도 함께 서술해줘.
  예: ["descript: 역사적 의미가 있는 장소로, 고풍스러운 건축 양식과 함께 봄에는 벚꽃이 아름답게 피어나는 명소입니다.", "descript: 관광객뿐만 아니라 현지인들도 자주 찾는 산책 코스로 유명해요."]

- "operationInfo":
  운영 정보를 리스트(String 배열)로 작성해줘. 다음 항목 반드시 포함:
    - runtime: 운영 시간 (하절기/동절기 구분이 있다면 명시)
    - holiday: 정기 휴무일 또는 특별 휴무일
    - fee: 입장료 (무료 여부 또는 세부 요금)

- "contactInfo":
  문의처 정보를 리스트(String 배열)로 작성해줘. 공식 웹사이트나 전화번호를 우선적으로 찾아보고, 없으면 null 값. 제발 공식적인 값만 넣어줘.
  무리하게 찾아보지말고, 없으면 그냥 null값 넣어줘.
  예: ["phone number: "02-123-4567", "웹사이트: www.example.com"]
  예시 내용은 절대 그대로 쓰지마. 참고용이야. 예시를 그대로 써야되는 상황이면 null 값으로 처리해줘.

- "info":
  관광지의 개요 및 핵심 정보로, 장소의 성격, 특징, 방문 시 유의사항, 대표 볼거리, 주변 환경 등
  다양한 내용을 종합적으로 서술해줘. 분량이 많을수록 좋고, 정보가 많을 경우 자연스럽게 나눠서 상세히 설명해줘.
  문단 2개로 나눠서 각 문단마다 많은 내용을 담아줘.
  각 문단마다 글자수는 1000자를 맞추려고 노력해줘. 분량을 정말 많이 넣어줘.

- "tip":
  여행 팁을 리스트(String 배열)로 제공해줘.
  혼잡 시간대, 사진 찍기 좋은 시간, 동선 추천, 피해야 할 시간, 꿀팁, 어린이/노약자 동반 시 유용한 정보 등을 다양하게 담아
  최대한 풍부하게 작성해줘.

- "history":
  관광지의 역사와 배경을 스토리텔링 형식으로 작성해줘.
  건축 연도, 역사적 사건, 유명 인물과의 관련성 등 단순 연표가 아닌, 재미있는 이야기처럼 풀어내는 게 좋고,
  전설, 일화, 관련 문화유산이나 기록 등도 포함해줘. 분량은 길수록 좋아.
  역사에 관련된 각각 다른 이야기를 2개 풀어줘. 각각 다른 문단으로 구분해서.
  각 문단마다 글자수는 1000자를 맞추려고 노력해줘. 분량을 정말 많이 넣어줘.

- "restaurants":
  관광지 주변 또는 내부에 있는 맛집 정보를 리스트로 작성해줘. 각 아이템은 아래 필드를 포함해야 해:
    - "name": 식당 이름
    - "description": 추천 이유와 대표 메뉴, 분위기, 이용 팁 등
    - "distance": 거리 정보 (도보 몇 분 / 차량 몇 분 등)

- "photoSpots":
  포토 스팟들을 리스트(String 배열)로 제공해줘.
  장소 이름만 나열하지 말고, 구체적인 위치와 어떤 장면이 잘 나오는지도 함께 설명해줘.
  최소 5개 이상은 추천해줘.

- "amenities":
  관광지 내 편의시설 정보를 리스트(String 배열)로 작성해줘.
  화장실, 수유실, 음수대, 휠체어 대여소, 유모차 대여소, 카페, 쉼터 등 방문객 편의를 위한 시설들을 최대한 많이 명시해줘.

절대 지켜야 할 규칙:
무조건 순수 JSON 형식으로만 응답해야 해.
JSON 외의 텍스트, 코드블럭, 설명, 인사말, 부연설명은 절대 포함하지 마.
가장 중요한 점 : 값이 null인 경우는 실제 null 값으로 처리해줘
json 구조는 다음과 같습니다.
{
  "title": String,
  "address": String,
  "description": [String],
  "operationInfo": [String],
  "contactInfo": [String] or null,
  "info": String,
  "tip": [String],
  "history": String,
  "restaurants": [
    {
      "name": String,
      "description": String,
      "distance": String
    }
  ],
  "photoSpots": [String],
  "amenities": [String]
}
